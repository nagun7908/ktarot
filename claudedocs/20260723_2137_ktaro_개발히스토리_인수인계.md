## 5. 핵심 기술 설계 — 로컬 해석 엔진 (런타임 LLM 0회)

무료 콘텐츠에 실시간 LLM을 붙이면 유저가 늘수록 적자. 바이럴이 목표라 **"무료 = 로컬 해석(원가 0원), 유료만 실시간 LLM"** 이 정답.

### 콘텐츠 3계층
- **Tier 1 (무료 · 런타임):** 사전 제작된 해석 DB 조회 + 템플릿 조합. 변동비 ~0원.
- **Tier 2 (제작 파이프라인):** 여기서만 LLM을 **배치로** 사용. LLM 초안 대량 생성 → 전문가 감수 → 금칙어 린트 → DB 적재. "LLM은 공장에서만, 매장에서는 안 쓴다."
- **Tier 3 (유료 · 실시간 LLM):** 결제된 심층 채팅 리딩 전용.

핵심 원칙: **"생성은 제작 시점(오프라인 배치), 서빙은 조회."** LLM을 안 쓰는 게 아니라, 유저 요청 순간이 아니라 콘텐츠 만드는 순간에 미리 쓰는 것.

### 엔진 작동 방식
- **결정론적 오늘의 카드:** `hash(사용자ID/닉네임 + 날짜)` 시드 → 같은 날 다시 열어도 같은 카드 (운세 일관성).
- **해석 DB 규모:** 78장 × 정/역 × 주제 = 코어 해석. 최종적으로 **7개 도메인 × 78장 = 3,276셀**까지 손저작 완결.
- **슬롯 조합:** `[공감 인트로] + [카드 연출] + [코어 해석] + [주제별 조언] + [다정한 마무리]`.

### v3에서 확정된 "말 이어짐" 개선 (중요)
초기엔 원자 슬롯을 런타임에 무작위로 이어붙여 문장이 끊기고 "복붙 느낌"이 났음. 해결책 3가지:
1. **질문을 상태값으로:** 주제 → 세부질문 → 대상을 선택으로 캡처해 `reading context` 객체로. 카드선택 · 분기 · 풀이 내내 상단에 **질문 핀** 상시 노출 + 중간 되짚기 문장 삽입.
2. **연결어 레이어 + 밸런스 정합:** 문장 뱅크에 태그(밸런스 긍/부/중, 위치 과거/현재/미래, 역할 오프닝/이음/마무리). 카드 사이에 관계 기반 접속어 자동 삽입("게다가/그리고" vs "다만/한편"). → **`BANK.connector`에 이미 저작돼 있었으나 조립부가 안 쓰던 것이 '말 안 이어짐'의 핵심 원인이었음.**
3. **head 트래픽 통짜 프리컴퓨트:** 연애 · 재회 · 이직 등 고빈도 `(주제 × 스프레드 × 밸런스패턴)` 조합은 유한하므로, 제작 시점에 LLM으로 **문단 단위 완성형 풀이**를 대량 생성 → 감수 → DB 적재. 런타임엔 변수 2~3개만 채움. 롱테일만 원자 슬롯 엔진으로 커버.

## 6. 카드 디자인 — "달빛 민화" 컨셉

- **팔레트:** 밝은 오방색 재해석 8색.
- **소재 매핑:** 메이저 22장 전체를 전통/민화 소재로 (해·달·연인·힘 등 → 달토끼, 원앙, 까치호랑이). 궁정 카드는 아이·나그네·마님·대감으로 한국식 재해석.
- **무서운 카드 순화:** Death → 환절기(지는 꽃 아래 돋는 새순), Devil → 미혹(구미호). "다정한 타로" 포지셔닝과 정합.
- **프롬프트팩:** 모든 카드에 붙는 [마스터 스타일 프롬프트]로 화풍·팔레트·"무섭지 않게" 규칙 고정. 메이저는 개별 장면, 마이너 56장은 [수트 베이스 4종] × [숫자·궁정 장면표 14행] 조합식. 수트 = 붓/찻잔/부채/엽전.
- **실전 규칙:** ① 글자(카드명·번호·테두리)는 AI에게 그리게 하지 말고 피그마 템플릿에서 일괄로 얹기. ② 대표 카드(해·달·힘)로 스타일 앵커(예: Midjourney `--sref`) 고정 후 나머지 생성.

## 8. 심층 검토 결과 (보고서 v1) — 보강할 부분

4축 진단: 사용자요구 **B−**, 상업성 **B**, 사용성 **C+**, 타로 해석 이론·임상 **C−**.

가장 약한 곳은 **축 4(타로 해석의 실체).** 시장·포지셔닝·수익구조는 A급인데 "타로 서비스로서의 실체"가 얇음. 구체적으로:
- 오리지널 덱을 만들면서 **상징 사전(모티프→의미) 없음**
- **원소적 위엄(카드 상호작용) 없음** → 3카드 스프레드가 사실상 3개 독립 운세
- **역방향 방법론 미선언**
- 융 심리학 프레이밍은 톤만 차용
- **임상·위기 대응 가드레일이 리스크표 한 줄뿐**

경쟁 벤치마크 8축 중 ktaro가 확실히 이긴 칸은 **구독 윤리(다크패턴 반대) 하나뿐.** 개인화·저널/후속·해석 투명성·AI 멀티모달은 결여(다만 대부분 Phase 2~3에 흩어져 있어 우선순위 재배치로 상당 부분 보완 가능).

**핵심 권고:** 출시 전 「ktaro 타로 해석 이론서 v1」을 별도 산출물로 제작. 이게 곧 "타로 전문성" 포지셔닝의 실체이자 감수의 기준.

## 9. 다음 우선순위 (합의된 방향)

**타로 이론/임상/조언+개운법을 쪼개지 말고 하나의 파운데이션 문서로 묶기.** (개운법은 임상·윤리 리스크가 가장 큰 조각이고 리딩 output의 마지막 조각이라 강령과 분리 불가.)

문서 구성:
1. **해석 이론** — 달빛 민화 상징 사전, 구조 이론(수비학·원소·코트), 역방향 방법론 선언, 스프레드 종합 규칙
2. **심리 프레이밍** — 융 기반 "성찰 도구"
3. **조언·개운법 설계 원칙** — 리딩 output 3단 구조(**해석→조언→개운법**), 개운법을 "예측적 부적"이 아닌 **"리추얼"로 프레이밍**(CHANI 방식, 굿즈 커머스 P2의 이론적 근거), 주제별 개운법 뱅크
4. **임상·윤리 강령** — 범위 제한(의료/법률/임신 예언 금지), 위기 리퍼럴 프로토콜, 금칙어 근거

**스코프:** MVP는 메이저 22장 먼저(전체 78장 아님) → 개발 병행 가능, M1 일정 유지.
**미결정 사항:** 개운법 톤을 어디까지 잡을지 — 순한 자기돌봄 리추얼만 vs 굿즈 연계까지.
